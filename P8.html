Платформы и технологии e-learning

Короткий экзаменационный конспект: LMS, CMS, LCMS, SCORM, xAPI, cmi5, LTI, Scrum, Kanban, Web 2.0, Web 3.0, mobile apps и выбор технологии под учебную цель.

1. LMS, CMS и LCMS — не путать

Главная тема файла — технологическая архитектура e-learning. Важно различать три типа систем: LMS управляет обучением, CMS управляет веб-контентом, LCMS помогает создавать и переиспользовать учебный контент.

Система Что это Главные функции Примеры
LMS Learning Management System — система управления обучением. Курсы, студенты, задания, оценки, прогресс, коммуникация, аналитика. Moodle, Canvas, Google Classroom, Blackboard, Brightspace.
CMS Content Management System — система управления веб-контентом. Страницы сайта, публикации, медиа, меню, пользователи, плагины. WordPress, Drupal, Joomla, Ghost, Webflow.
LCMS Learning Content Management System — система управления учебным контентом. Создание интерактивных модулей, учебных объектов, тестов, экспорт SCORM/xAPI. Articulate Rise, Storyline, Captivate, Xyleme.
Формула для экзамена: LMS = управляет обучением; CMS = управляет сайтом; LCMS = создает и переиспользует учебный контент.

Что важно помнить

LMS нужна, если надо отслеживать студентов, оценки, задания, прогресс и коммуникацию.

CMS подходит для образовательного сайта, блога, портала или страницы с материалами, но сама по себе не является полноценной системой оценки.

LCMS полезна для профессиональной сборки интерактивных e-learning-модулей и экспорта в стандартах совместимости.

Технология не заменяет педагогику: платформа должна обслуживать learning objectives, UX, доступность и оценивание.

2. Standards: SCORM, xAPI, cmi5 и LTI

Стандарты нужны для совместимости: чтобы e-learning-модуль можно было запускать в разных системах, отслеживать действия студента, передавать результаты и подключать внешние инструменты.

Стандарт Смысл Главное применение
SCORM Классический стандарт упаковки e-learning-контента для LMS. Запуск модуля в LMS + tracking: completion, score, progress.
xAPI Experience API фиксирует учебные действия как statements. Отслеживание learning experiences не только внутри LMS, но и в apps, websites, simulations.
cmi5 Профиль, который соединяет xAPI с логикой LMS. Более современная связь между гибким tracking и LMS-запуском.
LTI Learning Tools Interoperability. Подключение внешних учебных инструментов к LMS без ручного переноса данных.
Самое вероятное в тесте: SCORM и xAPI — это стандарты interoperability, которые позволяют отслеживать и передавать данные об обучении.

3. Scrum и Kanban

В e-learning-проекте нужно управлять контентом, дизайном, видео, интерактивностью, доступностью, тестированием, интеграцией и документацией. Для этого используются методы управления проектами.

Scrum

  • Agile framework для итеративной разработки.
  • Работа делится на sprints.
  • Есть backlog, sprint planning, review и retrospective.
  • Подходит для командной работы и проектов с изменяющимися требованиями.

Kanban

  • Visual management system.
  • Доска задач: To Do, In Progress, Review, Done.
  • Показывает workflow и статус работы.
  • Хорошо подходит для учебного проекта, потому что простой и наглядный.
Формула: Scrum = работа спринтами; Kanban = визуальная доска потока задач.

4. Web 2.0, Web 3.0, mobile apps и интерактивность

Эти инструменты расширяют e-learning за пределы обычной LMS. Они нужны для совместной работы, коммуникации, персонализации, мобильной практики и активного обучения.

Группа Смысл Примеры
Web 2.0 Совместное создание, обсуждение и публикация контента. Wiki, Padlet, Miro, Flipgrid, Notion.
Communication tools Коммуникация и координация группы. Discord, Microsoft Teams, Slack.
Web 3.0 Семантические технологии, AI, персонализация и predictive analytics. Educational chatbots, virtual tutors, predictive analytics.
Mobile apps Обучение через смартфоны: практика, повторение, microlearning. Kahoot!, Duolingo, Codecademy Go, Elevate.

Интерактивность

Quiz проверяет понимание и дает automatic feedback.

Simulation позволяет безопасно исследовать ситуацию или процесс.

Drag & drop помогает классифицировать, сопоставлять или упорядочивать элементы.

Scenario-based learning заставляет студента принимать решения и видеть последствия.

Feedback должен объяснять ошибку, а не просто писать “правильно/неправильно”.

Правило: интерактивность должна быть связана с learning objectives. Если клик не помогает учиться, это просто decoration.

5. Как выбрать технологию

Технологию выбирают не по модности, а по учебной задаче, аудитории, UX, доступности, tracking и необходимости интеграции.

Задача Что выбрать Почему
Управлять студентами, заданиями, оценками и прогрессом LMS Есть roles, gradebook, tracking, communication и analytics.
Создать публичный образовательный сайт CMS или custom website Удобно публиковать страницы, медиа, новости и навигацию.
Собрать интерактивный учебный модуль LCMS / authoring tool Есть templates, quizzes, interactions, SCORM/xAPI export.
Интегрировать модуль в LMS и отслеживать результат SCORM / xAPI / cmi5 / LTI Нужны interoperability и tracking.
Организовать разработку проекта Kanban / Scrum Помогает отслеживать задачи, версии, review и progress.

6. Что точно запомнить к тесту

LMS: система управления обучением: курсы, студенты, задания, оценки, коммуникация, прогресс.

CMS: система управления веб-контентом: страницы, публикации, медиа, сайт.

LCMS: система создания и переиспользования учебного контента и интерактивных модулей.

SCORM/xAPI: стандарты совместимости и tracking учебных данных.

Kanban: визуальное управление workflow через доску задач.

Scrum: agile-разработка через sprints, review и continuous improvement.

Web 2.0: collaborative tools: wiki, Padlet, Miro, Notion.

Web 3.0: AI, semantic technologies, personalization, predictive analytics, decentralization.

Mobile apps: подходят для microlearning, практики, повторения и быстрых quizzes.

Главный вывод: технология должна поддерживать learning objectives, UX, accessibility, assessment и analytics.

Типовые вопросы

Для чего нужен WordPress?
Для создания и управления контентом на websites. Это CMS.

Что характеризует Articulate Rise 360?
Это LCMS / authoring tool для создания e-learning content.

Что такое SCORM и xAPI?
Стандарты interoperability, которые помогают отслеживать и передавать learning data.

Что делает Kanban?
Визуально управляет потоком задач и оптимизирует процессы.

Что включает Web 3.0?
Семантические технологии, AI, персонализацию и децентрализацию данных.
