1. Что изучается в этом файле

TP10 продолжает технологический блок курса и детально объясняет стандарты интеграции e-learning-контента: SCORM, xAPI, cmi5 и LTI. Также файл затрагивает принципы интерактивности, активные стратегии, Web 2.0, communication tools, Web 3.0 и mobile apps.

Главная мысль: качественный e-learning-модуль должен не только выглядеть как сайт, но и быть совместимым с платформами, отслеживаемым, интерактивным, педагогически осмысленным и пригодным для интеграции в цифровую образовательную экосистему.

SCORM

Классический стандарт упаковки, запуска и отслеживания e-learning-контента в LMS.

xAPI

Гибкий стандарт для записи learning experiences в разных контекстах, не только внутри LMS.

cmi5

Профиль xAPI для LMS, который задает правила запуска, авторизации и мониторинга.

LTI

Стандарт подключения внешних образовательных инструментов к LMS через безопасную интеграцию.

Interactivity

Интерактивность должна поддерживать цели обучения, autonomy, feedback и active participation.

Web tools

Web 2.0, communication tools, Web 3.0 и mobile apps расширяют e-learning-среду.

2. Историческое развитие цифровых платформ

Презентация показывает развитие e-learning-платформ через три этапа: доминирование web, появление standards/open-source и современную фазу ресурсов, интеграции, cloud, mobile devices, analytics, personalization и AI.

Web

Курсы стали доступны через browser, постепенно заменяя CD-ROM. Это расширило аудиторию и привело к первой генерации коммерческих web-based LMS, которые управляли users, content, assessment и communication.

2000–2002

Появляется первая версия SCORM в 2000 году. В 2002 году появляется Moodle как первый open-source LMS, что усиливает роль standards и open-source в e-learning.

Современный этап

Cloud и mobile devices меняют пользовательские потребности. Возникают advanced analytics, xAPI, personalization и AI как новые направления развития цифрового обучения.

Логика развития: сначала важно было просто запускать курс в браузере. Потом стало важно переносить курс между платформами. Сейчас важно отслеживать learning experience в разных средах и персонализировать обучение.

3. SCORM — Shareable Content Object Reference Model

SCORM — это набор технических спецификаций, созданный для обеспечения interoperability между e-learning-контентом и разными платформами. Он позволяет образовательным материалам повторно использоваться и запускаться в нескольких LMS без специальных адаптаций.

Короткое определение: SCORM — это стандарт упаковки, запуска, коммуникации и отслеживания e-learning-контента внутри LMS.

Почему SCORM важен

  • До SCORM контент, созданный для одной платформы, часто плохо работал на другой.
  • Отсутствие общего стандарта увеличивало зависимость от конкретных поставщиков software.
  • Повторное создание одного и того же курса под разные системы повышало development costs.
  • SCORM создал общий способ packaging, communication и tracking.
  • Организации получили более стабильный сбор данных о progress, completion и performance.
  • SCORM стал доминирующим стандартом распространения digital learning content более чем на десятилетие.

Как работает SCORM

ZIP package

Курс распространяется как ZIP-пакет со всеми файлами и metadata.

imsmanifest.xml

Главный индекс курса: structure, resources и navigation rules.

JavaScript API

Контент общается с LMS через стандартизированную JavaScript API.

Tracking

Обычно фиксируются completion status, score, time и learner location.

Authoring tool

Контент создается в authoring tool и затем экспортируется в SCORM.

LMS execution

Совместимый LMS запускает пакет и принимает данные о прохождении.

Эволюция и наследие SCORM

  • SCORM 1.2 стал самой широко принятой версией из-за simplicity, stability и compatibility.
  • Главное наследие SCORM — interoperable ecosystem для распространения educational content.
  • Стандартизация tracking позволила сравнимо отслеживать completion, progress и performance.
  • Ограничения SCORM в mobile learning, offline learning и data model привели к развитию xAPI и cmi5.

4. xAPI — Experience API

xAPI, первоначально известный как Project Tin Can, был создан для сбора и хранения данных о learning experiences в разных контекстах. Он опубликован в 2013 году как ответ на новые потребности digital learning.

Короткое определение: xAPI фиксирует образовательный опыт не только в LMS, но и в mobile apps, simulations, games, presencial environments и других средах.

Почему xAPI стал нужен

  • Рост mobile apps, serious games, simulations и VR потребовал более гибкого tracking.
  • Стало нужно отслеживать опыт обучения на разных devices и contexts, не только в browser и LMS.
  • Организации захотели richer indicators: engagement, performance, collaboration, informal learning.
  • xAPI может фиксировать online и offline activities.
  • xAPI помогает строить lifelong learning records.
  • Он преодолевает LMS-centered tracking limitations традиционных моделей.

Как работает xAPI

Actor

Кто совершил действие: студент, сотрудник, пользователь.

Verb

Что сделал: completed, answered, watched, attempted, passed, shared.

Object

Над чем было действие: lesson, video, quiz, simulation, activity.

Context

Дополнительная информация: duration, date, result, source validation, environment.

LRS

Learning Record Store собирает, хранит и предоставляет learning records.

Analytics

Данные используются для reports, personalization, learning analytics и predictive analysis.

Преимущества xAPI

  • Гибкость: можно отслеживать почти любой learning experience.
  • Поддержка online и offline learning с последующей синхронизацией.
  • Интеграция LMS, mobile apps, enterprise systems и analytics tools.
  • Развитие Learning Record Stores, reporting tools и specialized profiles.
  • Поддержка personalization и data-based learning.
Пример xAPI statement: “Student completed Quiz 1” или “Student watched Introduction Video”. Любое xAPI-событие строится по схеме: actor + verb + object.

5. cmi5 — профиль xAPI для LMS

cmi5 — это профиль xAPI, созданный специально для LMS-сред. Он задает стандартизированные правила для launch, authentication и monitoring of educational content.

Короткое определение: cmi5 соединяет гибкость xAPI с традиционной LMS-логикой запуска и отслеживания курса.

Зачем нужен cmi5

  • xAPI очень гибкий, но без дополнительных правил может быть слишком свободным для LMS.
  • cmi5 задает согласованные правила запуска и monitoring контента в LMS.
  • Он не заменяет xAPI, а уточняет его применение для LMS-distributed courses.
  • Его развитие связано с ограничениями SCORM в mobility, authentication и restrictive data models.

Как работает cmi5

cmi5.xml

Описывает course structure, available modules и launch rules.

Assignable Units

AUs — учебные единицы, которые общаются с LMS через xAPI.

LMS launch

LMS передает identity, LRS link и secure authentication parameters.

Lifecycle statements

AUs отправляют данные: started, completed, passed, failed, terminated.

LRS records

Записи хранятся в LRS и используются для progress и results.

LMS decision

LMS читает данные из LRS, чтобы определить course completion и learner status.

Преимущества cmi5

  • Более строгая структура, чем у чистого xAPI.
  • Гибче SCORM, потому что основан на xAPI.
  • Подходит для distributed content, simulations и adaptive modules.
  • Лучше учитывает authentication, launch и LMS reporting.
  • Стратегически важен как мост между классической LMS-моделью и современным learning analytics.

6. LTI — Learning Tools Interoperability

LTI — стандарт 1EdTech для интеграции внешних applications и services с learning platforms, например LMS. В отличие от SCORM и xAPI, LTI не фокусируется на содержании или records of learning, а на связи между независимыми системами.

Короткое определение: LTI позволяет запускать внешние образовательные инструменты прямо из LMS через безопасную интеграцию и Single Sign-On.

Как работает LTI

  • LTI связывает Tool Consumer и Tool Provider.
  • Tool Consumer — принимающая платформа, например LMS.
  • Tool Provider — внешний инструмент: simulator, videoconference system, assessment platform, virtual lab.
  • Пользователь нажимает ссылку в LMS для доступа к внешнему инструменту.
  • LMS отправляет secure request с credentials и contextual data.
  • Могут передаваться name, email, user role, course context и grade information.
  • Новые версии поддерживают Deep Linking, grade synchronization и automatic participant management.

Почему LTI важен

  • Дополняет SCORM, xAPI и cmi5.
  • SCORM распространяет content, xAPI records experiences, а LTI соединяет systems and tools.
  • Позволяет строить modular educational ecosystems.
  • Снижает development costs и уменьшает зависимость от proprietary integrations.
  • Повышает technological flexibility и поддерживает innovation.
  • Является одним из основных механизмов interoperability между LMS и specialized services.

7. Сравнение SCORM, xAPI, cmi5 и LTI

Стандарт Главный фокус Где работает Что фиксирует / делает Главное ограничение
SCORM Упаковка и запуск контента. Внутри LMS. Completion, score, time, location. Ограничен LMS, browser, online-сценариями и жесткой data model.
xAPI Запись learning experiences. LMS, apps, games, simulations, offline/online contexts. Actor-Verb-Object statements + context/result. Нужен LRS и более сложная архитектура данных.
cmi5 xAPI в LMS-контексте. LMS + LRS. Launch, authentication, AU lifecycle, completion. Менее распространен и требует совместимой инфраструктуры.
LTI Интеграция внешних tools. LMS + external applications. Secure launch, SSO, contextual data, grade sync. Не является стандартом упаковки контента или полноценного learning record.
Коротко: SCORM — курс как пакет. xAPI — опыт как данные. cmi5 — xAPI для LMS. LTI — внешние инструменты внутри LMS.

8. Принципы интерактивности

Интерактивность в e-learning должна быть педагогически осмысленной. Пользовательское действие должно помогать understanding, application или consolidation of knowledge.

Alignment

Каждое действие пользователя должно быть связано с learning objectives.

Autonomy

Студент получает control, choice и возможности exploration.

Decision-making

Хорошая интерактивность заставляет выбирать, пробовать, сравнивать и применять.

Immediate feedback

Студент должен быстро понимать последствия своих действий и исправлять ошибки.

Meaningful feedback

Feedback должен объяснять, почему ответ правильный или неправильный.

Appropriate challenge

Сложность должна соответствовать уровню компетенции студентов.

Эффективная интерактивная среда

Наиболее эффективные digital interactive environments объединяют active participation, meaningful feedback, intuitive navigation и challenges, соответствующие уровню студентов.

Проверочный вопрос: помогает ли это действие студенту понять, применить или закрепить знание? Если нет — это не учебная интерактивность, а декоративный клик.

9. Активность “Operação Engagement”

В файле есть практическая активность на 60 минут: студенты в группах по 2–3 человека получают документ об эргономике и должны переработать его в мультимедийный ресурс, который будет более attractive и interactive для target audience.

Что нужно представить

  • Разработанную concept idea.
  • Основные proposed interactions.
  • Применение principles of interactivity.
  • Пример active participation.
Важно: технически разрабатывать ресурс не нужно. Цель активности — спроектировать learning experience, а не готовый продукт.

10. Plataformas Web 2.0

Web 2.0-платформы поддерживают collaborative learning, shared production, discussion и active participation. Они особенно полезны, когда студент должен не просто потреблять материал, а создавать, комментировать, сравнивать и сотрудничать.

Платформа Для чего подходит Учебная польза
Wiki Совместное создание и редактирование knowledge base. Collective construction of knowledge, documentation, glossary.
Padlet Визуальная доска с заметками, ссылками, изображениями и идеями. Brainstorming, sharing resources, быстрый сбор идей.
Miro Collaborative whiteboard для схем, maps, workflows и design thinking. Group planning, concept maps, storyboard, process visualization.
Flipgrid Видеоответы и короткие student presentations. Oral expression, reflection, peer feedback, social presence.
Notion Notes, databases, project documentation и shared workspace. Organization, collaborative documentation, project management.

Риск Web 2.0

Если не задать правила, роли и цель активности, Web 2.0 превращается в хаотичный обмен ссылками. Платформа должна поддерживать конкретную learning activity.

11. Ferramentas de Comunicação

Communication tools поддерживают синхронную и асинхронную коммуникацию, group coordination, feedback и accompaniment в online learning.

Discord

Каналы, voice rooms, community interaction, быстрые обсуждения и неформальная поддержка.

Microsoft Teams

Видеоконференции, файлы, классы, assignments, интеграция с Microsoft 365.

Slack

Командные каналы, threads, integrations, project communication и professional workflow.

Как использовать без хаоса

  • Определить, где задаются вопросы по контенту, а где обсуждается проект.
  • Разделить каналы по темам.
  • Закрепить инструкции и дедлайны.
  • Не дублировать одно и то же в нескольких каналах.
  • Использовать notifications осознанно, чтобы не создавать overload.

12. Web 3.0 в e-learning

Web 3.0 в контексте курса связан прежде всего с intelligent systems: educational chatbots, virtual tutors и predictive analytics. Эти инструменты позволяют усиливать personalization, сопровождение и data-based decision-making.

Educational chatbots

Отвечают на частые вопросы, помогают ориентироваться в модуле и дают базовую поддержку.

Virtual tutors

Могут сопровождать студента, предлагать подсказки и адаптировать объяснения.

Predictive analytics

Использует данные для выявления риска dropout, слабых мест и потребности в intervention.

Риски Web 3.0

  • AI может давать неточные или неподходящие ответы.
  • Нужны privacy, data protection и прозрачность использования данных.
  • Студент не должен полностью заменять критическое мышление автоматической подсказкой.
  • Predictive analytics может быть biased, если данные неполные или неправильно интерпретированы.

13. Mobile Apps

Mobile apps поддерживают learning on the move, microlearning, repetition, gamification и быстрые интерактивные проверки. Но мобильный формат требует коротких блоков, clear UI и responsive design.

Приложение Функция в обучении Что можно взять для своего проекта
Kahoot! Игровые quizzes и быстрая проверка понимания. Короткие вопросы, соревновательность, immediate feedback.
Duolingo Microlearning, streaks, gamification, короткие задания. Progress, небольшие уроки, повторение, видимая мотивация.
Codecademy Go Практика программирования и повторение материала на телефоне. Мини-упражнения, закрепление, learning by doing.
Elevate Тренировка cognitive skills через короткие упражнения. Короткие challenge-based activities и персонализированный прогресс.
Для e-learning: mobile не означает “тот же сайт, только маленький”. Нужны короткие блоки, крупные touch targets, простая навигация и быстрое действие.

14. Частые ошибки

Ошибка 1: путать standards: SCORM не то же самое, что xAPI, cmi5 или LTI.

Ошибка 2: считать SCORM устаревшим и бесполезным. Он до сих пор широко используется.

Ошибка 3: думать, что xAPI работает только в LMS. Наоборот, он нужен для разных контекстов.

Ошибка 4: забывать, что xAPI требует LRS.

Ошибка 5: воспринимать cmi5 как замену xAPI. Это профиль xAPI для LMS.

Ошибка 6: считать LTI стандартом упаковки контента. LTI — это integration между systems/tools.

Ошибка 7: добавлять интерактивность ради кликов, а не ради learning objectives.

Ошибка 8: использовать много communication tools без правил.

Ошибка 9: переносить длинный desktop-контент в mobile без адаптации.

Ошибка 10: не объяснить на защите, почему выбранная технология подходит именно твоему модулю.

15. Что важно запомнить к экзамену

Мини-конспект

eLearning standards: нужны для interoperability, tracking, reuse и integration.

SCORM: ZIP package + imsmanifest.xml + JavaScript API + LMS tracking.

SCORM records: completion, score, time, learner location.

SCORM legacy: interoperable ecosystem и стабильный LMS-tracking.

xAPI: records learning experiences через Actor-Verb-Object statements.

LRS: Learning Record Store хранит xAPI statements.

cmi5: xAPI profile для LMS launch, authentication и monitoring.

LTI: secure integration external tools with LMS, включая SSO и context data.

Interactivity: alignment with objectives, autonomy, exploration, decision-making, immediate feedback.

Web 2.0: Wiki, Padlet, Miro, Flipgrid, Notion.

Communication tools: Discord, Microsoft Teams, Slack.

Web 3.0: chatbots, virtual tutors, predictive analytics.

Mobile apps: Kahoot!, Duolingo, Codecademy Go, Elevate.

Вопросы для самопроверки

1. Для чего нужен SCORM?
Для упаковки, запуска, повторного использования и отслеживания e-learning-контента в LMS.

2. Что находится внутри SCORM-пакета?
ZIP с файлами курса, metadata и imsmanifest.xml, который описывает структуру и навигацию.

3. Чем xAPI отличается от SCORM?
SCORM в основном работает внутри LMS; xAPI может фиксировать learning experiences в разных средах.

4. Что такое Actor-Verb-Object?
Структура xAPI statement: кто сделал, что сделал и с каким объектом.

5. Что такое LRS?
Learning Record Store — система хранения и предоставления learning records.

6. Что такое cmi5?
Профиль xAPI для LMS, который задает правила запуска, авторизации и monitoring.

7. Что такое Assignable Unit?
Учебная единица в cmi5, которая запускается из LMS и передает xAPI statements.

8. Для чего нужен LTI?
Для безопасного подключения external tools к LMS, включая SSO и передачу контекста.

9. Что отличает хорошую интерактивность?
Она связана с objectives, дает autonomy, decision-making и meaningful immediate feedback.

10. Что такое Operação Engagement?
Активность, где группа перерабатывает документ в более привлекательный и интерактивный multimedia learning resource.

11. Как Web 3.0 применяется в e-learning?
Через chatbots, virtual tutors и predictive analytics.

12. Главный вывод TP10?
Технологии e-learning должны обеспечивать совместимость, tracking, integration, interactivity и pedagogical value.

Финальный вывод: TP10 показывает техническую архитектуру современного e-learning: SCORM отвечает за пакет и LMS-tracking, xAPI — за широкий learning record, cmi5 — за xAPI внутри LMS, LTI — за подключение внешних инструментов, а интерактивность и Web tools превращают контент в активный учебный опыт.
