eLearning Standards, интерактивность и цифровые инструменты

Сокращённый конспект без воды: SCORM, xAPI, cmi5, LTI, интерактивность, Web 2.0, Web 3.0, communication tools и mobile apps.

1. Зачем нужны eLearning Standards

Стандарты e-learning нужны для interoperability: чтобы учебный модуль можно было запускать на разных платформах, переиспользовать, отслеживать прогресс, передавать оценки и подключать внешние инструменты.

Главная идея: стандарт делает курс не просто сайтом, а учебным объектом, который может работать внутри LMS и передавать данные о прохождении.

SCORM

Классический стандарт упаковки, запуска и отслеживания e-learning-контента внутри LMS.

xAPI

Фиксирует learning experiences в разных средах: LMS, apps, simulations, games, offline/online.

cmi5

Профиль xAPI для LMS: задаёт правила запуска, авторизации и мониторинга.

LTI

Интеграция внешних учебных инструментов в LMS через безопасный запуск и SSO.

2. SCORM

SCORM — Shareable Content Object Reference Model. Это стандарт, который позволяет упаковать курс и запустить его в LMS с отслеживанием результата.

Как выглядит: обычно ZIP-пакет с файлами курса.

Главный файл: imsmanifest.xml — описывает структуру, ресурсы и навигацию курса.

Связь с LMS: через JavaScript API.

Что отслеживает: completion, score, time, learner location/progress.

Почему важен: один и тот же курс можно запускать в разных совместимых LMS.

Запомнить для теста: SCORM = пакет курса + запуск в LMS + tracking прогресса и оценки.

3. xAPI

xAPI, или Experience API, нужен для отслеживания учебного опыта не только внутри LMS. Он может фиксировать действия в мобильных приложениях, симуляциях, играх, сайтах, VR/AR-средах и даже offline-активностях.

Формула xAPI: Actor + Verb + Object. Например: “Student completed Quiz 1”.

Actor

Кто сделал действие: студент, пользователь, сотрудник.

Verb

Что сделал: completed, answered, watched, passed, failed.

Object

С чем работал: quiz, video, lesson, simulation, task.

LRS

Learning Record Store хранит xAPI statements.

xAPI гибче SCORM, но технически сложнее: обычно нужен LRS и продуманная структура данных.

4. cmi5 и LTI

cmi5

cmi5 — это профиль xAPI для LMS. Он сохраняет гибкость xAPI, но добавляет правила, нужные LMS: launch, authentication, Assignable Units, lifecycle statements и completion.

  • Не заменяет xAPI, а уточняет его для LMS.
  • Гибче SCORM.
  • Работает через LMS + LRS.
  • Подходит для современных модулей, simulations и adaptive content.

LTI

LTI — Learning Tools Interoperability. Это стандарт для подключения внешних инструментов к LMS: симуляторов, видеоконференций, assessment platforms, virtual labs и других сервисов.

  • Tool Consumer = LMS.
  • Tool Provider = внешний инструмент.
  • Поддерживает secure launch и Single Sign-On.
  • Может передавать user role, course context и grades.
Коротко: cmi5 = xAPI для LMS. LTI = подключение внешних tools к LMS.

5. Главное сравнение стандартов

Стандарт Главный смысл Где работает Что помнить на экзамен
SCORM Упаковка и tracking курса. Внутри LMS. ZIP + imsmanifest.xml + JavaScript API + completion/score/progress.
xAPI Запись learning experiences. LMS, apps, games, simulations, offline/online. Actor-Verb-Object statements, хранение в LRS.
cmi5 xAPI в LMS-логике. LMS + LRS. Launch, authentication, Assignable Units, completion.
LTI Интеграция внешних tools. LMS + external applications. Secure launch, SSO, context data, grade sync.
Самая короткая формула: SCORM — курс как пакет. xAPI — опыт как данные. cmi5 — xAPI для LMS. LTI — внешний инструмент внутри LMS.

6. Интерактивность в e-learning

Интерактивность нужна не ради кликов, а ради обучения. Хорошее действие пользователя помогает понять, применить или закрепить знание.

Alignment

Интерактивность связана с learning objectives.

Autonomy

Студент получает выбор, контроль и возможность исследовать.

Decision-making

Студент принимает решения, сравнивает и применяет знания.

Immediate feedback

Система быстро показывает результат действия.

Meaningful feedback

Объясняет, почему ответ правильный или неправильный.

Appropriate challenge

Сложность соответствует уровню студентов.

Проверка: если интерактивность не помогает learning objective, это decoration.

7. Web 2.0, Web 3.0, communication tools и mobile apps

Группа Смысл Примеры Учебная польза
Web 2.0 Совместное создание, обсуждение и публикация. Wiki, Padlet, Miro, Flipgrid, Notion. Collaboration, shared notes, brainstorming, group projects.
Communication tools Коммуникация и сопровождение обучения. Discord, Microsoft Teams, Slack. Group support, project coordination, synchronous/asynchronous communication.
Web 3.0 Интеллектуальные и персонализированные среды. Educational chatbots, virtual tutors, predictive analytics. Personalization, automatic support, early warning, adaptive guidance.
Mobile apps Обучение через smartphone/tablet. Kahoot!, Duolingo, Codecademy Go, Elevate. Microlearning, repetition, quizzes, learning on the move.

Риск Web 2.0: без цели и правил получается хаотичный обмен ссылками.

Риск communication tools: слишком много каналов создаёт overload.

Риск Web 3.0: AI может ошибаться, нужны privacy и критическое мышление.

Риск mobile: нельзя просто сжать длинный desktop-контент; нужны короткие блоки и простая навигация.

8. Частые ошибки

Ошибка 1: путать SCORM, xAPI, cmi5 и LTI.

Ошибка 2: думать, что SCORM и xAPI делают одно и то же.

Ошибка 3: забывать, что xAPI требует LRS.

Ошибка 4: считать cmi5 заменой xAPI. Это профиль xAPI для LMS.

Ошибка 5: считать LTI стандартом упаковки контента. LTI подключает внешние инструменты.

Ошибка 6: делать интерактивность ради кликов, а не ради learning objectives.

Ошибка 7: использовать много инструментов без правил и структуры.

Ошибка 8: переносить длинный desktop-контент в mobile без адаптации.

9. Что точно запомнить к экзамену

Мини-конспект

Standards: нужны для compatibility, reuse, tracking и integration.

SCORM: ZIP package, imsmanifest.xml, JavaScript API, LMS tracking.

xAPI: Actor-Verb-Object statements; records learning experiences; нужен LRS.

cmi5: профиль xAPI для LMS launch, authentication и completion.

LTI: secure integration external tools with LMS, SSO, context data, grade sync.

Interactivity: должна иметь цель, действие студента и meaningful feedback.

Web 2.0: Wiki, Padlet, Miro, Flipgrid, Notion — collaboration.

Web 3.0: chatbots, virtual tutors, predictive analytics.

Mobile apps: Kahoot!, Duolingo, Codecademy Go, Elevate — microlearning и practice.

Вопросы как в тесте

1. SCORM и xAPI — это что?
Стандарты interoperabilidade/tracking для e-learning.

2. Что фиксирует SCORM?
Completion, score, time, learner progress/location внутри LMS.

3. Чем xAPI отличается от SCORM?
xAPI отслеживает опыт обучения в разных средах, не только в LMS.

4. Что такое LRS?
Learning Record Store — место хранения xAPI statements.

5. Что такое cmi5?
Профиль xAPI для LMS.

6. Что такое LTI?
Стандарт подключения внешних learning tools к LMS.

7. Что делает интерактивность качественной?
Связь с objectives, autonomy, decision-making и meaningful feedback.

8. Web 3.0 в e-learning — это что?
AI, personalization, chatbots, virtual tutors, predictive analytics.

Финальный вывод: TP10 — это про техническую совместимость и активное обучение: SCORM упаковывает курс, xAPI записывает опыт, cmi5 связывает xAPI с LMS, LTI подключает внешние инструменты, а интерактивность превращает контент в действие студента.
